# CS405 – Secure Coding
This course focused on identifying, analyzing, and mitigating common software security vulnerabilities through secure coding practices, static code analysis, and security policy development. Throughout the course, I learned how to recognize security risks, remediate vulnerable code, and apply industry best practices to develop secure applications.

## Projects Completed

### Security Policy Development

Developed a comprehensive organizational security policy outlining security controls, user responsibilities, risk management strategies, and best practices for protecting software systems and sensitive information.

### Buffer Overflow Analysis

Analyzed vulnerable C/C++ code to identify buffer overflow vulnerabilities. Implemented secure coding techniques and proper bounds checking to eliminate memory safety issues and reduce the risk of exploitation.

### SQL Injection Prevention

Examined applications vulnerable to SQL injection attacks and implemented secure programming practices, including input validation and parameterized queries, to prevent unauthorized database access.

### Static Code Analysis

Performed static code analysis using automated security scanning tools to identify software vulnerabilities, distinguish legitimate findings from false positives, and remediate security weaknesses while maintaining application functionality.

### Secure Code Review

Conducted manual code reviews to identify insecure programming practices, improve code quality, and strengthen application security. This project reinforced the importance of reviewing code beyond automated scanning results.

## Learning Outcomes

This course strengthened my understanding of secure software development and reinforced the importance of incorporating security throughout the software development lifecycle. Through hands-on projects involving vulnerability analysis, secure coding, and security policy creation, I gained practical experience identifying common software vulnerabilities, implementing effective mitigations, and documenting security best practices. These projects improved my ability to develop software that is more secure, maintainable, and resilient against common cyber threats.
